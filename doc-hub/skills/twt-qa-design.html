QA · qa

/twt-qa-design

Audit source files for design fidelity — token-only CSS and section structure matching the system.

Version 1.1.2 Arguments: yes

Purpose

Checks your source files for design fidelity: styling that comes only from tokens (no stray colors or sizes), every token actually defined, and each page's structure including the components its layout calls for. It only reports — it doesn't edit.

Parameters it can expect

Optional local path; a URL is rejected (source-only audit)

What it reads

  • site/assets/css/The site's stylesheets.
  • site/The built static website.
  • .twt-artifacts/design/mockup/styles.cssThe shared stylesheet powering the mockups, built from design tokens.
  • .twt-artifacts/design/design-system/tokens.cssThe design tokens as CSS custom properties, ready for the build to consume.
  • .twt-artifacts/design/component/components.mdDetailed specs for every UI component — anatomy, variants, states, and tokens.
  • .twt-artifacts/design/layout/layouts/Per-page layout specs — section order, components, content, and breakpoints.

Result it provides

  • .twt-artifacts/qa/design-report.mdThe design & token fidelity audit findings.

Boundaries

  • Local source only. A served URL can't reveal token-only authoring; if given a URL, write a one-line note and exit
  • Doesn't edit anything (read-only)
  • Doesn't audit content or accessibility (other skills)

Success criteria

  • Writes .twt-artifacts/qa/design-report.md opening with a weighted Scorecard → Health (0–100) / Band (Pass ≥80 / Revise 50–79 / Fail <50), followed by BLOCKER / WARNING / SUGGESTION findings, each as Where / Problem / Recommendation
  • Flags every raw literal in token-only CSS and every undefined custom property reference
