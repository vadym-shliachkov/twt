/twt-audience

One-call audience workflow: define → validate in one pass (§9 — no iteration loop) — personas seeded from positioning segments plus journey stages that IA, curation, layout, and text...

Version 1.0.1 Arguments: yes

Purpose

Defines who the site is really for — named personas seeded from your positioning, each with the journey stages they move through — then reviews the result in the same pass. IA, curation, layouts, and text analysis all read these personas, so pages speak to real people instead of a generic visitor.

Parameters it can expect

Optional; runs define then the single validate pass

What it reads

  • .twt-artifacts/pre-design/audience/personas.mdA Markdown document.
  • .twt-artifacts/pre-design/audience/validation-report.mdA read-only critique of this artifact, listing blockers, warnings, and suggestions.

Result it provides

No files of its own — it orchestrates other skills or reports back to you.

Boundaries

  • Dispatches sub-skills (rule 5); no inline logic
  • No unbounded loop, no auto-downgrade of severity

Success criteria

  • Produces/refines personas.md + current validation-report.md
  • Honors the §9 single-pass policy: one define + one validate (folded into define under orchestration), at most one BLOCKER-driven re-run, no score-chasing loop; reports final Band + Health and surfaces open decisions per §13 (or bubbles them up in collect mode)
  • On exit, states whether BLOCKERs remain
