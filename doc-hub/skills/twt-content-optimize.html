/twt-content-optimize

Score your copy, rewrite it for clarity and brevity, then show the before/after improvement.

Version 1.2.2 Arguments: yes

Purpose

Makes a piece of writing better for you: it scores the original, rewrites it to be clearer and tighter, then shows the before-and-after with the reason behind each change. Let it rewrite everything at once, or walk through the suggestions one by one.

Skills it runs, in order

This skill is an orchestrator — it dispatches the skills below (via the Agent tool) rather than doing their work itself. Here's each one, in the order it runs, and why.

  1. /twt-content-validateScores the original copy first, so the improvement is measured rather than guessed.

Parameters it can expect

Optional subject (file path or pasted text); optional mode (auto|manual) and level (light|standard|aggressive)

What it reads

  • the subject textThe text you want analyzed — a file, pasted text, or an existing content artifact.
  • .twt-artifacts/content-quality/content-config.mdA Markdown document.
  • .twt-artifacts/content-quality/validation/<subject-slug>/validation-report.mdA read-only critique of this artifact, listing blockers, warnings, and suggestions.
  • .twt-artifacts/pre-design/brand/brand-brief.mdYour canonical brand brief: voice, tone, personality, and the visual cues the whole project follows.

Result it provides

  • .twt-artifacts/content-quality/subject/<subject-slug>.mdA Markdown document.
  • .twt-artifacts/content-quality/optimized/<subject-slug>.mdA Markdown document.
  • .twt-artifacts/content-quality/validation/<subject-slug>/validation-report-before.mdA Markdown document.
  • .twt-artifacts/content-quality/validation/<subject-slug>/validation-report-after.mdA Markdown document.
  • .twt-artifacts/content-quality/optimization-report.mdA Markdown document.
  • .twt-artifacts/content-quality/content-config.mdA Markdown document.
  • the subject file in placeYour original text file, updated in place — only with your explicit consent.

Boundaries

  • Doesn't invent facts, change business logic, or alter numbers/dates/prices/legal wording without explicit instruction
  • Doesn't replace the source file silently — in-place writes require explicit consent; the optimized copy always lands in .twt-artifacts/content-quality/optimized/ first
  • Doesn't re-implement the rubric — scoring is dispatched to /twt-content-validate (rule 5)

Success criteria

  • optimization-report.md shows a before/after Scorecard delta (per criterion and total Health) plus a change log explaining each edit and the criterion it serves
  • In manual mode, nothing is applied before the user decides (keep / apply all / apply selected / apply + switch to auto), via AskUserQuestion
  • "Switch to auto" persists content_review_mode: auto to .twt-artifacts/content-quality/content-config.md and later runs honor it
  • Guardrails hold: meaning, facts, and legal/compliance wording survive verbatim unless the user explicitly allowed changes
