QA · qa

/twt-qa

Run the right QA audits on local files or a live site, then report a pass/fail verdict and punch-list.

Version 1.0.6 Arguments: yes

Purpose

Runs the right quality checks for your project — on local files or a live site — and rolls them up into a pass/fail report plus a plain-language list of what's left to fix. It's the final gate before you call the site done.

Skills it runs, in order

This skill is an orchestrator — it dispatches the skills below (via the Agent tool) rather than doing their work itself. Here's each one, in the order it runs, and why.

  1. /twt-qa-contentChecks every page exists and matches the content plan.
  2. /twt-qa-designChecks styling is token-only and structure matches the design system.
  3. /twt-qa-a11yChecks accessibility basics — alt text, headings, labels, contrast.
  4. /twt-qa-linksChecks links resolve and responsive tiers hold up.
  5. /twt-qa-elementorChecks Elementor theme code hygiene — only for WordPress builds.

Parameters it can expect

Optional http(s):// URL (live mode) or local path; else local auto-detect

What it reads

  • .twt-artifacts/qa/content-report.mdThe content & IA fidelity audit findings.
  • .twt-artifacts/qa/design-report.mdThe design & token fidelity audit findings.
  • .twt-artifacts/qa/a11y-report.mdThe accessibility audit findings.
  • .twt-artifacts/qa/links-report.mdThe link-integrity and responsive-tier audit findings.
  • .twt-artifacts/qa/elementor-report.mdThe Elementor code-hygiene audit findings.

Result it provides

  • .twt-artifacts/qa/qa-report.mdThe rolled-up QA verdict — pass/fail across every audit that ran.
  • .twt-artifacts/qa/gaps.mdA plain-language punch-list of everything left to fix.

Boundaries

  • Doesn't reproduce audit logic — dispatches each audit via the Agent tool (rule 5)
  • Doesn't auto-fix anything — reports and stops; the human resolves BLOCKERs
  • Doesn't do live performance/pixel-render checks (out of scope)

Success criteria

  • Runs the audits applicable to the mode and writes qa-report.md (verdict PASS iff total BLOCKERs == 0) and gaps.md
  • qa-report.md frontmatter records mode, url (if live), verdict, targets, and skipped audits
  • gaps.md lists every LOREM / EMPTY / MISSING-ASSET / DEAD-LINK / PLACEHOLDER-LINK item grouped by page
