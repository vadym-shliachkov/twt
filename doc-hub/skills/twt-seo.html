General · seo

/twt-seo

One-call SEO workflow: define → validate in one pass (§9 — no iteration loop) — per-page keywords, slugs, meta drafts, schema, and the redesign redirect map.

Version 1.0.1 Arguments: yes

Purpose

Plans the site's search presence page by page — target keywords, slugs, meta titles and descriptions, schema, and (for redesigns) the redirect map — then reviews the plan in the same pass. Development gets concrete values to build in, instead of SEO being left as an afterthought.

Parameters it can expect

Optional; runs define then the single validate pass

What it reads

  • .twt-artifacts/pre-design/seo/seo-map.mdA Markdown document.
  • .twt-artifacts/pre-design/seo/validation-report.mdA read-only critique of this artifact, listing blockers, warnings, and suggestions.

Result it provides

No files of its own — it orchestrates other skills or reports back to you.

Boundaries

  • Dispatches sub-skills (rule 5); no inline logic
  • No unbounded loop, no auto-downgrade of severity

Success criteria

  • Produces/refines seo-map.md + current validation-report.md
  • Honors the §9 single-pass policy: one define + one validate (folded into define under orchestration), at most one BLOCKER-driven re-run, no score-chasing loop; reports final Band + Health and surfaces open decisions per §13 (or bubbles them up in collect mode)
  • On exit, states whether BLOCKERs remain
