/twt-qa-links

Audit links for integrity and check declared responsive tiers, flagging dead and placeholder links.

Version 1.1.2 Arguments: yes

Purpose

Makes sure every link and anchor actually resolves, navigation stays consistent, and your declared responsive breakpoints hold up. It flags dead and placeholder links so they land on your fix list.

Parameters it can expect

Optional local path or http(s):// URL; else auto-detect site/ then Phase-2 mockups

What it reads

  • site/The built static website.
  • site/partials/Shared HTML partials reused across every page.
  • .twt-artifacts/design/mockup/pages/The rendered HTML mockup for each page.
  • .twt-artifacts/design/assets/manifest.mdAn inventory of the images and assets the design references.

Result it provides

  • .twt-artifacts/qa/links-report.mdThe link-integrity and responsive-tier audit findings.

Boundaries

  • Doesn't edit anything (read-only)
  • Doesn't network-probe external links — lists them only
  • Responsive-CSS checks are local-only (no source CSS to inspect live)

Success criteria

  • Writes .twt-artifacts/qa/links-report.md opening with a weighted Scorecard → Health (0–100) / Band (Pass ≥80 / Revise 50–79 / Fail <50), followed by BLOCKER / WARNING / SUGGESTION findings, each as Where / Problem / Recommendation
  • Flags every dead internal link and placeholder link (these feed the wrapper's gaps.md)
