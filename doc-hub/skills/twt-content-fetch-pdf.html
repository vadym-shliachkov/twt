/twt-content-fetch-pdf

Convert a PDF's readable text into clean Markdown for brand, IA, and curation work.

Version 1.0.0 Arguments: yes

Purpose

Pulls the readable text out of a PDF and saves it as clean Markdown, ready for brand, IA, and curation work. Handy when your raw material lives in reports, brochures, or exported docs.

Parameters it can expect

Path to a local PDF file (or folder of PDFs)

What it reads

  • <pdf-path>The PDF you give it.

Result it provides

  • .twt-artifacts/pre-design/content-fetch/pdf/<filename>/index.mdThe cleaned Markdown text extracted from a PDF.
  • .twt-artifacts/pre-design/content-fetch/pdf/<filename>/_meta.mdNotes about the fetched source — where it came from and any extraction caveats.

Boundaries

  • Not OCR for scanned/image-only PDFs — text-layer extraction only (flag when a PDF appears image-only)
  • Doesn't preserve exact visual layout, columns, or pixel positions
  • Doesn't extract embedded images as files (notes their presence only)

Success criteria

  • Output appears under .twt-artifacts/pre-design/content-fetch/pdf/<filename>/
  • index.md has frontmatter (source path, title, fetched-at, page count)
  • Headings and lists are preserved where the PDF's text structure allows
