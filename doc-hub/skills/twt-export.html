/twt-export

Create polished exports — PDF, DOCX, or slides — from a single guided command.

Version 1.1.2 Arguments: yes Token usage Mid

Purpose

Makes polished, on-brand documents and decks straight from your Markdown — PDF, Word, or slides — all from one guided command. It asks what you want, sets up a template if you need one, then runs the right export.

Skills it runs, in order

This skill is an orchestrator — it dispatches the skills below (via the Agent tool) rather than doing their work itself. Here's each one, in the order it runs, and why.

  1. /twt-export-template-createSets up a reusable template first when you don't already have one.
  2. /twt-export-pdf or /twt-export-docx or /twt-export-presentationRuns the export you chose — PDF, Word, or slides.

Parameters it can expect

Optional export type, source Markdown path or source instructions, theme choice, aspect ratio, and force flag

What it reads

  • <markdown-path>The Markdown file you give it.
  • .twt-artifacts/export/themes/*/theme.jsonA JSON data file.
  • templates/themes/doc-hub-light/theme.jsonA JSON data file.
  • tools/export-source-create.mjsThe helper that prepares a Markdown source for export.
  • tools/export-document.mjsThe script that renders Markdown into DOCX/PDF documents.
  • tools/export-presentation.mjsThe script that renders Markdown into slide decks.
  • tools/export-theme-create.mjsA generated file.

Result it provides

  • .twt-artifacts/export/sources/<source-slug>.mdThe Markdown source prepared for export.
  • .twt-artifacts/export/sources/<source-slug>.notes.mdNotes captured about an export source.
  • .twt-artifacts/export/pdf/<source-slug>/<source-slug>.pdfThe finished, on-brand PDF document.
  • .twt-artifacts/export/pdf/<source-slug>/<source-slug>.htmlAn HTML page.
  • .twt-artifacts/export/docx/<source-slug>/<source-slug>.docxThe finished, on-brand Word document.
  • .twt-artifacts/export/presentation/<source-slug>/<source-slug>.pptxThe finished slide deck in PowerPoint format.
  • .twt-artifacts/export/presentation/<source-slug>/<source-slug>.pdfThe finished slide deck as a PDF.
  • .twt-artifacts/export/presentation/<source-slug>/<source-slug>.htmlAn HTML page.
  • .twt-artifacts/export/themes/<theme-slug>/theme.jsonA JSON data file.
  • .twt-artifacts/export/themes/<theme-slug>/css/*.cssA stylesheet.
  • .twt-artifacts/export/themes/<theme-slug>/fonts/*A generated file.
  • .twt-artifacts/export/themes/<theme-slug>/reference/*A generated file.
  • .twt-artifacts/export/themes/<theme-slug>/preview/preview.htmlAn HTML page.
  • .twt-artifacts/export/themes/<theme-slug>/preview-notes.mdA Markdown document.

Boundaries

  • Doesn't convert files directly inside the orchestrator
  • Doesn't create both document and presentation outputs unless the user explicitly runs separate exports
  • Doesn't overwrite sources, themes, or outputs without explicit user consent
  • Doesn't duplicate logic from /twt-export-pdf, /twt-export-docx, /twt-export-presentation, or /twt-export-template-create

Success criteria

  • With no $ARGUMENTS, uses AskUserQuestion to choose output type before asking for source and theme choices
  • Supports HTML, PDF, DOCX, PPTX, and PDF presentation formats, all using the built-in doc-hub-light theme by default
  • Allows either an existing Markdown file or new source instructions
  • Offers only themes relevant to the selected output type, plus the built-in default and a create-new option
  • If creating a theme, dispatches /twt-export-template-create first and then proceeds with the export
  • Delegates source creation to tools/export-source-create.mjs
  • Delegates final conversion to the relevant child export skill, which delegates to tools/export-document.mjs or tools/export-presentation.mjs
