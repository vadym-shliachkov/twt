/twt-marketplace-docs

Regenerate SKILLS.md, architecture.md, and the README table from every skill's frontmatter.

Version 1.0.3 Arguments: no

Purpose

Rebuilds the marketplace's reference docs — the skills index, architecture map, and README table — straight from each skill's definition. Run it after you change any skill so the docs never drift out of sync.

Parameters it can expect

(none)

What it reads

  • commands/*.mdEvery skill's command definition, read to regenerate the docs.
  • skills/*/SKILL.mdEvery sub-skill's definition, read to regenerate the docs.

Result it provides

  • SKILLS.mdThe auto-generated command reference for the marketplace.
  • architecture.mdThe auto-generated map of how the skills fit together.
  • README.mdThe repo README — only the auto-managed skills table inside it is updated.

Boundaries

  • Doesn't modify skill body content — only stamps the description: frontmatter field and rewrites generated files
  • Doesn't validate skill body content — only frontmatter
  • Doesn't create per-category README files (those no longer exist)
  • Doesn't enforce strict lint rules — only warns about missing required fields

Success criteria

  • SKILLS.md lists every skill found, sorted alphabetically, with the (vX.Y.Z) version stripped from the displayed description
  • architecture.md contains a current mermaid diagram of skill dependencies plus a per-skill detail table
  • Validation warnings printed for any skill missing required frontmatter fields
  • Every auto-generated file starts with the AUTO-GENERATED header
  • Root README.md marked block (if present) is updated
